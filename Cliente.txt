El cliente solicita una pagina que sea la que se encargue de vender los boletos de un lugar de eventos, principalmente conciertos, debe estar enfocada en conciertos, debe incluir varios requerimientos como por ejemplo:  
Que para el administrador del sitio le sea posible acceder con un login necesario,y que ya dentro del sistema le sea posible:
-Agregar eventos con la informacion requerida como:
	Nombre del evento,y que se puedan agregar precios.
-Editar eventos:
	Que el administrador pueda editar toda la informacion de un evento previamente guaradado.
-Ver eventos;
	Que el administrador pueda ver la informacion de boletos disponbiles, fechas o cualquier otra info de un evento guardado. 
-Eliminar evento.

Cada evento debe tener 3 tipos de boletos y estos ademas deben su precio unico, una fecha y un ID. 

Cabe mencionar que no se debe poder sobrevender el evento, (comprar boletos anque no haya disponibilidad) ni tampoco comprar el mismo asiento dos veces.

Como usuario este debe poder comprar un boleto de la seccion elegida, con mensajes de confirmacion si hubo error en la compra y especificar por que. Debe poder ver el mapa para solicitar el boleto y tambien debe haber una pantalla de confirmacion donde se avise que el usuario tiene un tiempo donde se reserve el boleto elgido.

Debe usar Mysql para guardar estos eventos.
